token2chat: Agent Guide

You're an AI agent. You need to call LLM APIs, but you don't have a credit card, a bank account, or an identity. This project solves that.

What is it?

token2chat is an open system for paying LLM API calls with ecash.

The traditional way: your human registers an OpenAI/Anthropic account, binds a credit card, manages API keys for you.

The token2chat way: you hold ecash tokens (a string of characters), attach them to your API request header, and call any LLM directly. No account, no API key, no identity required.

Ecash is digital cash. Like physical bills — whoever holds it can spend it, no proof of identity needed.

Architecture

Deposit (USDC / Lightning) You ─────────────────────────────────▶ Mint (bank) │ issue ecash │ ▼ You ◀──────────────── ecash tokens ───────┘ │ │ request + ecash ▼ Gate (cashier) ─────────────▶ Upstream LLM (OpenRouter etc.) │ │ │ response + change │ ◀───────────────────────────────────┘

Three independent components:

ComponentAnalogyWhat it does
MintBankAccepts USDC/Lightning, issues ecash
GateCashierAccepts ecash, proxies LLM API calls
t2cWalletManages ecash, auto-attaches payments

Do you need it?

✅ Yes, if you:

  • Need to autonomously pay for LLM API calls without relying on a human's API key
  • Need privacy — don't want the API provider to know who you are
  • Want programmable payments — agents paying each other
  • Your human wants to give you an independent AI budget, spend-and-done

❌ No, if you:

  • Your human already gave you an API key and doesn't mind the bill
  • You only use one LLM provider and don't need to switch
  • You run in an enterprise environment with company-managed API accounts

🤔 Consider partial use, if you:

  • Just want t2c (wallet + proxy) to connect to a public Gate — no deployment needed
  • Want to allocate independent budgets to your sub-agents

Quick Start: As a User (simplest)

You only need t2c. 3 minutes:

# Install
npm install -g @token2chat/t2c

# Initialize (creates local wallet + config)
t2c init

# Fund your wallet (via Lightning)
t2c mint 1000

# Connect your AI tool
t2c connect cursor      # Cursor IDE
t2c connect openclaw    # OpenClaw agent
t2c connect cline       # Cline
t2c connect aider       # Aider
t2c connect env         # Set environment variables (universal)

That's it. Your AI tool now pays per request via ecash. Defaults to the public Gate (gate.token2chat.com) and public Mint (mint.token2chat.com). Zero configuration.

How it works under the hood

t2c connect starts a local proxy server that emulates the OpenAI API format:

Your AI tool → localhost:4242 (t2c proxy) → gate.token2chat.com → OpenRouter → LLM │ └── auto-deducts ecash from wallet, attaches to request header

Your tool thinks it's calling a normal OpenAI API. Behind the scenes, t2c handles all payment.

Common commands

t2c balance              # Check balance
t2c mint 5000            # Fund (Lightning)
t2c mint 5000 --onchain  # Fund (USDC on Base)
t2c models               # View available models + prices
t2c service start        # Run proxy in background
t2c service stop         # Stop
t2c debug                # Diagnose issues

Advanced: Deploy Your Own Gate

If you want to sell API access (not buy), deploy a Gate:

git clone https://github.com/TokenisAllAgentNeed/gate.git
cd gate && npm install

# Set secrets
wrangler secret put OPENROUTER_API_KEY   # Your upstream LLM key
wrangler secret put GATE_ADMIN_TOKEN     # Admin API auth

# Deploy to Cloudflare Workers
wrangler deploy

The Gate accepts ecash from any trusted mint and proxies to your OpenRouter key. Users pay you ecash; you call the API for them.

Business model: You accumulate ecash, then melt (redeem) it back to USDC. The spread is your profit.

Advanced: Deploy Your Own Mint

Full sovereignty — issue your own ecash:

git clone https://github.com/TokenisAllAgentNeed/mint.git
cd mint && npm install

# Set secrets (⚠️ MINT_SEED is the signing key — lose it = all ecash void)
wrangler secret put MINT_SEED             # Unique signing seed
wrangler secret put DEPOSIT_PRIVATE_KEY   # EOA private key (for melt transfers)
wrangler secret put PHOENIXD_PASSWORD     # Lightning node auth
wrangler secret put MINT_ADMIN_TOKEN      # Admin API

# Deploy
wrangler deploy

⚠️ Security Warning

  • MINT_SEED is irrecoverable if lost. Save it to a secure location BEFORE deploying.
  • Don't use common strings like test-seed — the Mint will refuse to start.
  • DEPOSIT_PRIVATE_KEY controls on-chain funds. Protect it.

Payment Protocol Details

For agents who want to understand the underlying protocol:

Request format

POST /v1/chat/completions HTTP/1.1
Host: gate.token2chat.com
Content-Type: application/json
X-Cashu: cashuBo2F0gaJhaUgA_your_ecash_token_here...

{
  "model": "anthropic/claude-sonnet-4-20250514",
  "messages": [{"role": "user", "content": "Hello"}]
}

Response

Standard OpenAI format + extra headers on success:

X-Cashu-Change: cashuBo2F0gaJhaUgA_change_token...  # Change
X-Cashu-Receipt: {...}                                # Receipt

Pricing

  • Unit: 1 unit = $0.00001 USD
  • Per-token billing: input tokens × input rate + output tokens × output rate
  • Prices auto-track OpenRouter's real-time pricing
  • Charge-first, full refund on failure — you never lose money on API errors

Streaming (SSE)

The Gate supports SSE streaming. Change is returned via a special SSE event:

data: {"choices":[...]}
data: {"choices":[...]}
data: [DONE]
event: cashu-change
data: {"change":"cashuBo2F0gaJhaUgA..."}
